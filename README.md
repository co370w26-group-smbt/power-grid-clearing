## Usage
1. Install dependencies:
    ## macOS
    Open your terminal and run:
    ```bash
    curl -LsSf [https://astral.sh/uv/install.sh](https://astral.sh/uv/install.sh) | sh
    ```

    ## Windows (Using Powershell)
    ```powershell
    powershell -ExecutionPolicy ByPass -c "irm [https://astral.sh/uv/install.ps1](https://astral.sh/uv/install.ps1) | iex"
    ```

    ## Sync dependencies:
    ```bash
    uv sync
    ```

2. With dependencies installed, run demo.ipynb, to run LPs on the sample 181-bus system.

Extra: For exact formulation, see `da-clearance-lp` and `ftr_clearance`

## Project Structure

### raw-data
Contains the raw data used in our project.

#### `sf-contingencies.csv` — Shift Factor (PTDF) Matrix

Exported from PowerWorld. Each row represents a bus; each column beyond the first five is a monitor/contingency scenario.

| Column | Description |
|--------|-------------|
| `Bus Number` | Integer bus identifier |
| `Name` | Bus name |
| `Area Name` | Area/zone identifier |
| `ETLR` | Emergency transfer limit rating (normal conditions) |
| `WTLR` | Worst-case transfer limit rating (under contingency) |
| `<N> (MONITOR_<m>_<f1>-<t1>_CONTINGENCY_<c>_<f2>-<t2>)` | Shift factor (PTDF) for bus `N`: change in flow on monitored line `m` (buses `f1`→`t1`) per 1 MW injected at bus `N`, when line `c` (buses `f2`→`t2`) is out of service |

Column names are generated by `formulation-automation/power-world-contingency.ipynb`, which produces one monitor/contingency pair for every ordered pair of distinct branches (38 branches → 38×37 = 1,406 SF columns).

### formulation-automation
Contains code for automatically generating intermediate files used to solve main LP(s) described in formulation.pdf.
